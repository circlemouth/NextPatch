# UIスタイルガイド案（DADS適用）

## 基本方針

NextPatch の UI は DADS 参照ルールに基づくアクセシブルな管理画面スタイルとする。DADS が未整備としているコンポーネントは、NextPatch 独自仕様として断定せず、`NextPatch暫定ルール` として扱う。

## カラー

| 用途 | 初期案 | 注意 |
|---|---|---|
| 背景 | `#F8FAFC` | 文字とのコントラストを確認 |
| サーフェス | `#FFFFFF` | カード、パネル |
| 本文 | `#0F172A` | 4.5:1 以上 |
| 補助テキスト | `#475569` | 14px使用時は特に確認 |
| 枠線 | `#64748B` | 非テキスト 3:1 以上 |
| Primary | `#1D4ED8` | CTA、リンク |
| Primary hover | `#1E40AF` | 状態差を一貫 |
| Success | `#166534` / `#F0FDF4` | 完了、成功 |
| Error | `#B91C1C` / `#FEF2F2` | エラー、失敗 |
| Warning | `#92400E` / `#FFFBEB` | 注意 |
| Info | `#1D4ED8` / `#EFF6FF` | 情報 |

状態は色だけで伝えず、ラベル・アイコン・テキストを併用する。

|---|---:|---|
| space-1 | 8px | 小要素間 |

| Token | 値 | 用途 |
|---|---:|---|
| none | 0 | テーブル等 |
| sm | 4px | チップ、小ラベル |

| full | 9999px | ピル型タグ |

## エレベーション

- 原則フラット。
- 浮かせる要素はボーダーを併用。
- シャドウだけで境界を示さない。
- モーダルは DADS 未整備のため、MVP では原則避ける。

## アイコン

- アイコンは意味補助であり、単独で状態を表さない。
- 外部リンクには外部リンクアイコン + テキストを併用。
- アイコンボタンにはアクセシブルな名前を付ける。

## ボタン

| 種類 | 用途 | ルール |
|---|---|---|
| Primary | 画面の主操作 | 原則1画面1個 |
| Secondary | 次点操作 | 同一文脈で最大3個 |
| Tertiary | 戻る/キャンセル/補助 | 控えめに表示 |
| Destructive | 削除/破壊的操作 | 文言と確認を併用 |

- 押下対象は44 CSS px以上。
- PCでは進む操作を右、戻る/中止を左。
- スマホでは縦積み。Primary を上に置く。
- disabled は原則避け、押下後に具体的エラーを出す。

## フォーム

- 可視ラベル必須。
- `※必須` / `※任意` を表示。
- 入力条件や例はサポートテキストに書く。
- placeholder を説明代わりにしない。
- エラーは入力欄直下に `＊` から始めて具体的に書く。
- maxlength に頼らず、超過時に検証エラーを返す。
- コピー・ペースト禁止はしない。

## テーブル

- PC の構造化データ比較に使う。
- スマホではカードリストへ変換。
- ソート、ページング、列固定などの高度テーブルコントロールは `NextPatch暫定ルール`。

## カード

- スマホ一覧、ダッシュボード、未整理メモに使う。
- タイトル、種別、状態、次操作を初期表示。
- 補助情報だけディスクロージャー可。重要情報は隠さない。

## 通知

- ページ上部または対象セクション内にバナー表示。
- success/error/warning/info を分ける。
- 操作完了後は「元に戻す」導線を置ける。
- 固定表示する場合は閉じるパーツを用意。

## ナビゲーション

- PC: 左サイドナビ。
- スマホ: ハンバーガーメニュー + ドロワー。
- ボトムナビゲーションは使わない。
- モバイルメニューは2階層以内。

## レスポンシブ

| 幅 | 扱い |
|---:|---|
| 768px未満 | モバイル/タブレット。1カラム |
| 768px以上 | デスクトップ。2〜3カラム可 |

確認幅: 375, 390, 768, 1024, 1440px。

## アクセシビリティ

- WCAG 2.2 AA 相当を目標。
- キーボードのみで主要導線完了。
- フォーカスリングを見える状態で維持。
- リンクは色だけでなく下線等で識別。
- コントラスト: 通常文字 4.5:1、非テキスト 3:1 以上を目標。

## 禁止または原則回避するUI

- ボトムナビゲーション
- 重要情報をアコーディオン内だけに隠す
- placeholder に説明を依存する
- disabled で操作不能理由を隠す
- 複雑なモーダル入力
- タブで主要情報を隠す
- ドラッグ&ドロップだけに依存する
- 色だけの状態表示
