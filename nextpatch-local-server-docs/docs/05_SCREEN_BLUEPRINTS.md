# 画面設計図

各画面は DADS 参照ルールを優先し、重要情報を折りたたみの中に隠さない。フォームは可視ラベル、要否、サポートテキスト、具体的エラーを必須にする。

## ダッシュボード

- 目的: 全リポジトリ横断で「今やるべきこと」を把握する。
- 主な表示情報: 今やるべき、重大バグ、次バージョン対象、未整理メモ、最近完了、最近更新repo。
- 主な操作: 詳細表示、完了/修正済み、クイック登録、repoへ移動。
- PCレイアウト: 左ナビ + 中央セクション + 右サマリー。
- スマホレイアウト: 1カラム。クイック登録、今やるべき、重大バグ、未整理メモの順。
- 空状態: repo追加 CTA。
- エラー状態: DB/Auth取得失敗をページ上部バナー。
- DADS観点: 重要情報を初期表示。プライマリーCTAはクイック登録。
- 関連データ: repositories, work_items, bug_details, status_histories.
- 受け入れ条件: 今やる対象が理由付きで表示される。

## リポジトリ一覧

- 目的: repo を一覧・選択する。
- 主な表示情報: repo名、URL、未完了数、重大バグ数、未整理メモ数、最終更新。
- 主な操作: 追加、詳細、編集、アーカイブ。
- PCレイアウト: テーブルまたはカードグリッド。
- スマホレイアウト: カードリスト。
- 空状態: 「リポジトリを追加」CTA。
- エラー状態: GitHub URL解析失敗は入力欄直下。
- DADS観点: テーブルは比較用。高度テーブル操作は暫定案。
- 関連データ: repositories.
- 受け入れ条件: URLから owner/repo が抽出される。

## リポジトリ詳細

- 目的: repo 単位の作業基地。
- 主な表示情報: repo概要、現在の焦点、次アクション、未修正バグ、進行中、予定、アイデア、技術、参考、未整理メモ。
- 主な操作: WorkItem追加、状態更新、分類、技術/参考追加。
- PCレイアウト: 8:4 または 9:3 の2カラム。
- スマホレイアウト: 次アクション、重大バグ、クイック登録を先頭。
- 空状態: 「このリポジトリにはまだ項目がありません。」
- エラー状態: アーカイブ済み repo は復元導線付きで表示。
- DADS観点: タブに依存しない。見出しとセクション縦積み。
- 関連データ: repositories, work_items, tech_notes, reference_services.
- 受け入れ条件: 3操作以内に item 追加・完了・分類ができる。

## WorkItem一覧

- 目的: 全 WorkItem を検索・絞り込みする。
- 主な表示情報: タイトル、種別、状態、優先度、repo、期限、完了日時。
- 主な操作: 検索、フィルタ、状態更新、詳細。
- PCレイアウト: フィルタ + テーブル。
- スマホレイアウト: カードリスト。
- 空状態: 条件解除リンク。
- エラー状態: 検索失敗をセクション内バナー。
- DADS観点: フィルタはラベル付き。色だけで状態を示さない。
- 関連データ: work_items, tags.
- 受け入れ条件: repositoryId null の item も表示・検索できる。

## 未整理メモ

- 目的: ChatGPTメモや思いつきを分類する。
- 主な表示情報: タイトル、作成日、形式、機密度、候補件数、parse error。
- 主な操作: タスク化、バグ化、アイデア化、技術メモ化、参考化、アーカイブ。
- PCレイアウト: メモカード + 分類アクション。
- スマホレイアウト: 主要3操作を表示し、詳細分類は専用ページ。
- 空状態: 「未整理メモはありません。」
- エラー状態: JSON不正でも保存済みとして表示。
- DADS観点: 重要操作をメニュー奥に隠さない。
- 関連データ: work_items(type=memo), classification_candidates.
- 受け入れ条件: 元メモを破壊せず分類先を作れる。

## クイック登録

- 目的: 情報を失わず保存する。
- 主な表示情報: repo、種類、タイトル、本文、機密度、今やるチェック。
- 主な操作: 保存、キャンセル、ChatGPT標準プロンプトコピー。
- PCレイアウト: 専用フォームページ。
- スマホレイアウト: 本文入力を主にした1カラム。
- 空状態: なし。
- エラー状態: 本文未入力時は静的エラー。
- DADS観点: placeholder説明禁止、disabled回避。
- 関連データ: work_items.
- 受け入れ条件: repo未選択でも memo として保存できる。

## 技術メモ

- 目的: 採用技術・候補・却下理由を残す。
- 主な表示情報: 技術名、カテゴリ、採用状態、理由、公式URL、懸念。
- 主な操作: 登録、採用、不採用、廃止。
- PCレイアウト: 一覧 + 詳細。
- スマホレイアウト: カード。
- 空状態: 技術メモ追加CTA。
- エラー状態: URL不正を具体表示。
- DADS観点: 状態を色だけで伝えない。
- 関連データ: tech_notes.
- 受け入れ条件: candidate→adopted を記録できる。

## 参考サービス

- 目的: 参考サービス・競合・参考記事を整理する。
- 主な表示情報: 名前、URL、種別、参考ポイント、良い点、懸念点。
- 主な操作: 登録、アイデア化、採用候補化、アーカイブ。
- PCレイアウト: カード + フィルタ。
- スマホレイアウト: カードリスト。
- 空状態: 追加CTA。
- エラー状態: URL不正を具体表示。
- DADS観点: 外部リンクを下線とテキストで示す。
- 関連データ: reference_services.
- 受け入れ条件: URLと参考点を保存できる。

## 設定 / データ管理 / システム状態

- 目的: ローカル運用、export/import、状態確認。
- 主な表示情報: runtime mode、DB/Auth状態、app version、export最終日時、backup注意、SMTP状態。
- 主な操作: JSON export、Markdown/CSV export、JSON import検証、logout。
- PCレイアウト: セクション型。
- スマホレイアウト: 1カラム。
- 空状態: なし。
- エラー状態: export/import失敗をページ上部バナー。
- DADS観点: 機密情報警告を折りたたまない。物理削除は専用確認ページを優先。
- 関連データ: export_logs, import_jobs, system status.
- 受け入れ条件: 3操作以内で JSON export に到達できる。
