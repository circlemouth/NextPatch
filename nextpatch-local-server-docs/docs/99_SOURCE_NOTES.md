## ローカルサーバー版への主な変更

元の検討ではクラウド/Vercel/Supabase Cloudが主案に含まれていたが、ユーザー要望により MVP の実行基盤をローカルサーバー版へ変更した。

変更後の正本:

- Next.js App Router + TypeScript
- Supabase Local / Self-hosted Supabase
- PostgreSQL
- Docker Compose
- Supabase Auth
- JSON export 必須

## DADS 参照

`references/dads_app_ui_design_rules_20260411.md` を優先する。特に以下を NextPatch へ反映する。

- アクセシビリティを前提にする。
- サービス内で一貫性を保つ。
- 重要情報を隠さない。
- フォームはラベル、サポートテキスト、具体的エラーを持つ。
- placeholder を説明代わりにしない。
- disabled を安易に使わない。
- ボトムナビゲーションは使わない。
- DADS未整備部品は NextPatch暫定ルールとして扱う。
